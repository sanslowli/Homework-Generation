## 2. Syntax Bingo의 핵심 구조 — 빙고판 5×8 = 40

수강생이 한 사이클(4~5주)에 가지는 빙고판:

- **10 구문** (그 챕터의 회화 패턴 10개) × **4 페르소나** (수강생 본인 + 동료 3명) = **40 문장**
- 5 × 8 그리드로 시각화
- 자기 문장 10 + 동료 문장 30 = **자기 자신의 이야기만 외우는 게 아니라 동료 30 문장도 그리고 발화**하는 게 사회적 학습의 핵심

각 구문은 4명의 학생이 각자 자기 이야기로 채움 → 한 구문에 대해 4가지 변주.

## 3. 챕터 시리즈 분류

### 5XX vs 6XX

| 시리즈 | 특징 |
|---|---|
| **5XX** | 유튜브 촬영 스크립트 기반. 친구 간 티키타카에서 구문 선별 (*Friends* 시리즈 바이브). 도메인 넓음 (취미·루틴·취향 등 일반 대화). |
| **6XX** | 도메인 극단 축소. *"서울 직장인이 SLB(Sunday Local Brunch)에 참여하여 2시간 세션 or 끝나고 어울리며 할법한 상황"*으로 한정. 관계 거리 = 모르는 사이 ~ 중간 친함. |

5XX 문제의식 = *"배워도 쓸데가 없다"* → 6XX 해법 = *"다음 일요일에 바로 쓴다"* (SLB가 실전 무대).

### 심화 / 기초

- 챕터 코드 **숫자만** (예: `601`, `602`, `603`) = **심화**
- 챕터 코드 **숫자 + S** (예: `601S`, `602S`, `603S`) = **기초**

기초는 심화의 일부 문장을 단순화하거나 세포분열(한 문장을 두 칸으로 분리)한 버전.

심화/기초 페이지 분리: **예문 DB(노션)** 제목이 `602@이름`(심화) / `602S@이름`(기초)로 나뉨. 반면
**빙고판(챕터) DB**는 `602(S)` 한 페이지에 두 레벨 교재 PDF가 같이 달림(레벨 구분은 예문 DB 제목에서).

**핵심 룰 — 콘텐츠 레벨은 '담은 사람'을 따른다**: 어떤 동료 문장이 누구 보드에 들어갔을 때,
그 문장의 연습 레벨 = *그 보드 주인의 레벨* (문장 원작자의 레벨이 아님). 예: 기초 학생이 담은
심화 동료의 문장도 **기초로** 연습 → 조회 시 기초 예문(`602S@동료`)을 가져와야 맞음.

### 구간(섹션) 구조 (6XX 한정)

6XX 챕터는 12 구문을 의미 단위 *구간*으로 묶음. SLB 테이블 실제 대화 흐름 반영.

⭐ **구간 경계는 챕터마다 다르며, 노션 `빙고판(챕터)` DB**의 `구간1~4 시작칸 / 구간1~4 끝칸`
숫자 속성이 **권위 있는 원본**(구간4 시작칸이 비면 3구간 챕터). 대시보드·스크립트는 여기서 읽어야 함.
아래 표의 분할을 하드코딩하지 말 것.

| 챕터 | 구간 수 | 분할 (시작칸~끝칸) | 출처 |
|---|---|---|---|
| **601(S)** | 4 구간 | #1~3 / #4~6 / #7~9 / #10~12 | 2026-06-12 DB 확인 ✅ |
| **602(S)** | 4 구간 | #1~4 / #5~6 / #7~9 / #10~12 | 2026-06-12 DB 확인 ✅ |
| **603(S)** | 3 구간 | #1~5 / #6~8 / #9~12 | 2026-06-12 DB 확인 ✅ |

> 정정: 종전 문서의 "601,602 = #1~3/4~6/7~9/10~12" 중 **602는 부정확**. 실제 602는 구간1이 #1~4(4칸),
> 구간2가 #5~6(2칸)로 균등 분할이 아님(601·603은 종전 기재가 맞음). 챕터마다 다르니 항상 DB 속성을 읽을 것.
> 심화·기초는 같은 페이지(예 `602(S)`)라 구간 경계도 동일.

구간별 이름·설명 → `~/Kusuk HQ/kusukban/Correction/chapter-reference/` 폴더의 각 챕터 md 참조.

## 4. 파일 명명 규칙 ⭐

이 섹션이 가장 중요. AI가 코드 보고 직관적으로 알기 힘든 자리들.

### 빙고판 그림 파일: `{section}-{slot}.png`  (매칭되면 → `{section}-{slot}{주인}.png`)

예: `2-3.png`, 매칭 후 `2-3박대호.png`

- 첫 자리 = **구간 번호** (이 챕터의 몇 번째 구간인가, 1 ~ 4)
- 둘째 자리(slot) = **그 구간에서 몇 번째 '사람 자리'인가** (사람 슬롯) — 2026-06-12 San 확인 ✅
  - 보통 같은 slot 번호 = 같은 동료(예: `1-2`와 `2-2`는 평소 같은 사람의 콘텐츠).
  - 단 **보강·타반 참관** 시 특정 구간의 그 slot만 다른 사람(타반)으로 바뀜 → 그래서 slot↔사람이 고정이 아님.
  - **그 slot에 실제로 들어간 사람(주인) = ImageMatching 시트의 `ContentOwner` 가 권위 원본** (8장 참조).
- **매칭 완료 표기**: 매칭되면(웹앱 자동 or 선생님 수동) 파일명 끝에 주인 이름을 붙임 → `2-3박대호.png`.
  이름표 없는 맨이름 = **아직 미매칭**(폴더에서 육안 식별용). 파서(`extract_section_slot_from_filename`)는
  slot 뒤 한글 이름을 허용하고, 시트 키는 항상 맨이름 `2-3.png`로 정규화(`match_image_key`). (v0.8 changelog)
- **파일 경로**: `{Syntax Pitching | Syntax Only | Syntax + Open-ended Question}/{학생}/{현행 챕터 | 지난 챕터}/{챕터}/{파일}`
  → ImageStudent = 학생 폴더명, Chapter = **이미지의 부모 폴더명(=챕터)**. (app.py·`sync_imagematching.py` 동일 규칙)

### 학생 일별 인증 이미지: `{YYYY-MM-DD}_{학생명}.jpeg`

예: `2026-04-29_정민주.jpeg`

- 날짜 = 인증한 날짜 (그날의 DailyHomework 완료)
- 학생명 = 한국어 본명
- 이 이미지는 **단톡방 출석 루프용 카톡 인증 이미지** (앱이 자동 생성)

## 5. SyntaxPitching 앱의 핵심 도메인 룰

### DailyHomework — 매일 3장 출제

**0608 변경: 종전 10장 → 3장.** 절대 다시 10장으로 돌리지 마라 (San이 정한 핵심 룰).

구조: 현행 챕터 **2장** + 지난 챕터 **1장** = 총 3장.
한쪽이 부족하면 자동 보충.

### 출제 우선순위

1. **타율 낮은 그림** (최근 5회 기준)
2. **5회 미만 신규 그림**

→ 약점 보강 + 데이터 수집 동시 해결.

미통과 누를수록 다음 출제 확률 ↑ (수강생 기록 자체가 커리큘럼이 되는 구조).

### 커닝 페이퍼

수강생이 사전에 정답을 등록해두면, 화면 **꾹 누르기**로 노출. 발화 못 할 때 살짝 보고 다시 시도.

### 카톡 인증 이미지

DailyHomework 완료 시 자동 생성 → 단톡방 출석 루프.

- 단일 컬럼, 좌측 정렬
- 그림 240px 높이
- Batting Average 섹션 없음 (현행)

### 빙고판 40 문장 유효 기간

수업 1회/주 외 평일 루틴 유지 장치. **잊기 전 반복**이 핵심.

## 6. 오디오 위젯 구조 (3-Row Layout)

각 그림 슬롯의 버튼 배치 (위에서 아래로):

| Row | 버튼 | 색 | 역할 |
|---|---|---|---|
| 1 | 🎙️ Hold to Record | 빨강 | 누르고 있는 동안 녹음. 떼면 자동 재생. |
| 2 | 🔈 내 녹음 듣기 | 빨강 | 개별 슬롯의 내 녹음 재생 |
| 3 | ▶️ 정답 듣기 | 파랑 | 정답 음성 재생 (클릭만, 자동 X) |

추가:
- **🎙️ 합본 듣기** — 모든 슬롯의 내 녹음을 이어 재생 (정답이 아니라 본인 녹음)
- **Safari 오디오 unlock priming** — 첫 재생 시 무음 WAV로 element를 unlock하는 패턴. Safari per-element unlock 우회. 건드릴 때 주의.

### 자동 재생 타이밍

녹음 종료 후 ~250ms setTimeout으로 자동 재생. 옛 pre-warm(150ms) + autoplay timer(80ms) ≈ 230ms 결합치와 매칭.

**겹침/여집합 의미**: 여러 학생 보드를 비교할 때, 한 (주인·#번호) 콘텐츠를 *몇 개 보드가 공통으로 담았나*로
그룹화함. 같은 #번호라도 **레벨(심화/기초)이 다르면 다른 콘텐츠**(병합 금지) — 레벨은 위 '담은 사람' 룰을 따름.

## 10. SyntaxPitching 앱 vs SyntaxPitching™ 상품 — 헷갈리기 쉬움

같은 이름이지만 **다른 것**:

- **SyntaxPitching 앱** (이 폴더) = 정규 수강생 전용 **자가 복습 툴**. 무료 부속물.
- **SyntaxPitching™ 상품** = 1:1 프라이빗 영어 코칭 (경제력 O · 사회성 X 고객용). 별도 결제 상품. **현재 고객 X**.

코드에서 *"SyntaxPitching"* 단어가 나오면 앱을 의미. 상품 컨텍스트는 마케팅 본부(`~/Kusuk HQ/`) 영역.
